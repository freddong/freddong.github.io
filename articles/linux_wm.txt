## Linux下个人Window Manager使用习惯
首先前提是终端上用了多年的screen/tmux,在screen的习惯是每个编号对应1个工作区.

tmux的习惯:
- 0 => 机动用于临时性工作,比如安装软件,重启服务.
- 1-4 => 当前专注的项目的多个工作区,比如1用于项目的跑单元测试,2用于项目的运行http server,3用于看log,4个大多情况是够用的
其他新增窗口用于临时的工作,用完后Ctrl-D关闭

将这个习惯带到window manager后,也是给窗口编号,比如无论何时,按下Super-4就跳转/打开ide,按下Super-1就跳转浏览器,按下Super-3就跳转Slack. 注: super是windows键.

对窗口编号支持好的是StumpWM.

我当前的配置是.

0 => 终端
1 => Firefox
2 => Emacs 用于笔记
3 => Slack 沟通
4-5 => idea 开发可能多个项目

这些的前提是:
- 已经有固定的工作流程
- 这个工作流程是以项目为核心的, 打开的窗口都是为项目服务的
- 对于开发的话,其实打开的窗口(换句话说,用到的软件)是不多的,基本7个以内
- 用到的窗口少另一个原因是因为终端/浏览器/Emacs本身就是超级App,承载了很多功能
